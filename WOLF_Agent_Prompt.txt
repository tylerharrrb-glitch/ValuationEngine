SECTION 3 — WACC CALCULATION (Complete Specification)
3.1 — Formula
WACC = (E/V) × Ke + (D/V) × Kd × (1 − t)

Where:
E  = Market Capitalization (= Shares Outstanding × Current Stock Price)
D  = Total Debt (Short-Term Debt + Long-Term Debt)
V  = E + D
We = E / V   (equity weight)
Wd = D / V   (debt weight)
Ke = Cost of Equity (from CAPM — see Section 3.2)
Kd = Pre-tax Cost of Debt (user input, weighted average across all debt)
t  = Corporate Tax Rate (Egyptian default: 22.5%)

⚠️  NEVER use Book Equity for the WACC equity weight. Always use Market Capitalization. Book equity reflects historical cost; market cap reflects current investor expectations.

3.2 — Cost of Equity: Dual CAPM Methodology
Implement a toggle/dropdown letting the user select Method A (default) or Method B.

Method A — Local Currency CAPM (DEFAULT for EGP-denominated DCF)
Ke = Rf(Egypt) + β × Mature_Market_ERP
Egyptian Rf already embeds country risk and expected inflation. Therefore:
DO NOT add a separate Country Risk Premium when using Method A. That would double-count country risk and overstate Ke by ~7-10%.
Default values for Method A:
Rf: 10-year Egyptian Government Bond Yield (NOT the CBE overnight rate) — default 22.0%, display with date stamp
Mature Market ERP: 5.50% (Damodaran — updatable by user)
CRP field: HIDDEN when Method A is active. Display note: 'Country risk is embedded in the local risk-free rate. No CRP added.'

Method B — USD Build-Up (for USD-denominated analysis)

SECTION 4 — FCFF CALCULATION (Three-Method Cross-Verification)
4.1 — The Three Methods (ALL must be implemented)
Calculate FCFF simultaneously via all three methods. Display all three results side by side. If any method produces a different result (tolerance: ±0.001), highlight the discrepancy in RED and block the user from proceeding until it is resolved.

Method 1 — NOPAT Route (Primary)
FCFF = EBIT × (1 − t) + D&A − CapEx − ΔWC
     = NOPAT + D&A − CapEx − ΔWC

Method 2 — EBITDA Route
FCFF = EBITDA × (1 − t) + D&A × t − CapEx − ΔWC

Method 3 — Net Income Route
FCFF = Net Income + Interest × (1 − t) + D&A − CapEx − ΔWC

⚠️  FCFF must NEVER subtract Interest Expense directly. Interest cost is captured in WACC via the after-tax cost of debt. Double-counting interest would understate FCFF and enterprise value. If Method 3 is correct, adding back Interest×(1-t) neutralizes its effect.

⚠️  SIGN CONVENTION FOR ΔWC: An INCREASE in net working capital is a cash OUTFLOW and must be SUBTRACTED. Many engines get this backwards. Test: if Accounts Receivable increases, cash flow decreases.

4.2 — JavaScript Implementation
// Method 1
const nopat = ebit * (1 - taxRate);
const fcff1 = nopat + dAndA - capex - deltaWC;

// Method 2

// Cross-verify

SECTION 5 — DCF PROJECTIONS & TERMINAL VALUE
5.1 — Projection Mechanics
For each year t from 1 to N (default N=5, options: 3, 5, 7, 10):
Revenue_t   = Revenue_{t-1} × (1 + Revenue_Growth_t)
EBITDA_t    = Revenue_t × EBITDA_Margin_t
D&A_t       = Revenue_t × DA_Pct_t
EBIT_t      = EBITDA_t − D&A_t
NOPAT_t     = EBIT_t × (1 − Tax_Rate)
CapEx_t     = Revenue_t × CapEx_Pct_t
ΔWC_t       = (Revenue_t − Revenue_{t-1}) × DeltaWC_Pct
FCFF_t      = NOPAT_t + D&A_t − CapEx_t − ΔWC_t
Allow user to set different assumptions per year OR constant values across all years. Provide toggle: 'Use constant drivers' vs 'Set year-by-year drivers'.

5.2 — Terminal Value
Gordon Growth Method
TV = FCFF_N × (1 + g_terminal) / (WACC − g_terminal)

Soft warning: If g_terminal > 10%, show: 'Terminal growth exceeds 10%. For Egypt, long-term nominal GDP growth is typically 8-12%. Verify your assumption.'

Exit Multiple Method

SECTION 6 — DIVIDEND DISCOUNT MODELS
6.1 — Gordon Growth (Single-Stage)
P = D₀ × (1 + g) / (Ke − g)  =  D₁ / (Ke − g)
Validation: g < Ke (hard block). If company pays no dividends, show: 'DDM Not Applicable — Company does not pay dividends.'

6.2 — Two-Stage DDM
P = Σ[D₀(1+g₁)^t / (1+Ke)^t] for t=1..n  +  [Dₙ(1+g₂)/(Ke−g₂)] / (1+Ke)^n
Where g₁ = high-growth phase rate (e.g., 15%), g₂ = stable terminal rate (e.g., 8%), n = projection years.

6.3 — H-Model
P = D₀ × (1 + g_L) / (Ke − g_L)  +  D₀ × H × (g_S − g_L) / (Ke − g_L)
Where H = half-life of high-growth period in years. Models a gradual linear decline from g_S to g_L (more realistic than abrupt two-stage).

⚠️  ALL DDM models must discount at Ke (cost of equity), NOT WACC. DDM values equity directly. Discounting at WACC would undervalue the stock for leveraged companies.
